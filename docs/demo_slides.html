DiagramPlot · 數學核心

修正 5 類 公式 bug，每條都有 primary literature

項目舊版（錯）修正後文獻
Inverse isochron FF = 1/slope；LS 用 trapped 36/40 當 F（物理錯誤）F = −slope / intercept（York convention）Vermeesch 2024 / 2018
WMA 加權平均loop 內 1/σ² 互消 → 退化成 ΣTᵢ（純加總）Σ(T/σ²) / Σ(1/σ²)Schaen 2021
MSWD 參考點算術平均改用 WMASchaen 2021
σ_J 誤差傳播括號錯位 → σ_J 高估數個數量級(e^λt−1)/F_r² 正確偏導Bevington & Robinson 1992
σ₃₆ / σ₃₉corrections σ 被雙重計算從 raw component 扣回 σ²₃₆ₘ=σ²₃₆ₐ−σ²₃₆c…quadrature 一致化
這幾條不是我自己想的，是把程式對齊 IsoplotR / Schaen 的 community convention。

DiagramPlot · 影響範圍

這些 bug 影響每一份過去的結果

  • SH 樣品 Int age：舊版 F 用 1/slope → age 系統性偏低
  • LS 樣品 Int age：舊版 F 用 trapped 36/40 → 完全錯
  • 所有 WMA：舊版等同 ΣTᵢ，不是加權平均
  • 所有 MSWD：舊版用算術平均當參考點
  • 所有 σ_age：舊版 σ_J 高估 → σ_age 整條系統性高估

※ age 中心值修正後不變 的只有 σ_J 那條（只縮小誤差）。

Argon Pipeline · 流程
07
Argon Pipeline · ① Calculate T₀

智慧挑訊號，每顆 cycle 都看得到為什麼

  • Serial 挑 cycle：按物理依賴順序 Ar37→Ar36→Ar38/39/40，不再各 isotope 獨立亂挑
  • z-score 著色：MAD-based robust z，藍/琥珀/紅一眼看出 outlier，hover 顯示 t / mV / z
  • σ 雙軌可切換：pcov[-1,-1]（Li 2019）vs std(|r|)/√n，圖上同時印兩式
  • T₀ Range 盒鬚圖：blank box + 各 step T₀±σ，黃帶標目前 step，輔助挑 blank cycle

對老師強調：AutoPipeline 的 σ_T0 改 SE-from-covariance，但主程式 CalcT0Page 您指定的 std(|r|)/√n 我沒動。
